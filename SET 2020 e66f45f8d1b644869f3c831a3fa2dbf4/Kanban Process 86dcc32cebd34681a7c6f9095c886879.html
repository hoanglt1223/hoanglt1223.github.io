Cột 1 : Backlog

Chia công việc ra thành nhiều cái Tickets

DEV, SE không được quyền đụng vào

VD : Có 10 cái Tickets

Cột 2 : Todo

Ví dụ tính toán tuần này làm 8 cái Tickets → Lấy 8 cái Tickets từ Backlog sang

Engineer tự pick Ticket để đem qua In Progress làm - Một SE không được lấy quá 2 Tickets

Cột 3 : In Progress

Làm xong → Kéo qua Coding Done

Cột 4 : Coding Done

Project Manager sẽ cầm Tickets ở đây đem ra cột Review !

Khi làm xong bước Coding Done → PHẢI TẠO 1 cái MERGE REQUEST hoặc PULL REQUEST

Cột 5 : Fixing

Không giới hạn số Tickets

Khi có xuất hiện Tickets trong cột Fixing thì phải DỪNG ở CỘT In Progress → Luôn ưu tiên Fixing trước so với In Progress

Cố gắng Hạn chế bị trả về

Khi Fixing xong → Developer sẽ kéo ngược về Coding Done

Cột 6 : Review

Nếu PM thấy không đạt yêu cầu → Trả về cột Fixing !

Nếu PM thấy ổn → Kéo qua Ready !

Cột 7 : Ready

QA QC sẽ kéo qua cột Test để Test

Cột 8 : Testing

QA QC thấy không đạt yêu cầu → Trả về Fixing

QA QC thấy đạt → Kéo qua Release

Cột 9 : Done

Cột 10 : Release
