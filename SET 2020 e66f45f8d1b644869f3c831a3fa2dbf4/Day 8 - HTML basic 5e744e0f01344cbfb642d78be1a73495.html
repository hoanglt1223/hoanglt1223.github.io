Hình rất to ⇒ độ phân giải thực của hình ⇒ tốn băng thông khi load web về

  • Cấu trúc cơ bản HTML:
<!DOCTYPE html> khai báo đây là file HTML
<html> can't delete bc chứa nội dung ⇒ bỏ đi ko còn là file HTML
<head> chứa thông tin (meta data) ⇒ bỏ đi ko thấy chuyện gì xảy ra
<title></title>
</head>
<body> chứa nội dung ⇒ bỏ đi mất hết
<!-- văn bản và các thẻ TAG -->
</body>
</html>
  • Thành phần chính của file HTML

1. Trên cùng có khai báo <!DOCTYPE HTML>: để thông báo đây là 1 file HTML

⇒ Lí do: ngày xưa còn nhiều loại trang web ko chỉ riêng HTML

2. Thẻ <head> chứa thông tin (meta data) ⇒ bỏ đi ko thấy chuyện gì xảy ra

3. Thẻ <body> chứa nội dung ⇒ bỏ đi mất hết

4. Thẻ <html> ko đc bỏ chứa nội dung ⇒ bỏ đi ko còn là file HTML

Note: có 1 số tag chỉ có tag mở ko có tag đóng. Lí do: ko chứa tag nào nữa bên trong

⇒ Đặc điểm: có dấu gạch chéo ở sau, VD: <br/>.

trong là inner element

<input/> ⇒ 1 cục đó vẫn là element

<input id= "..." class = "abc"/>

in đậm: attributes: ⇒ key = "value"

  • Các tag có màu đỏ ⇒ ko còn đc support bởi HTML5
  • Sửa type "password" ⇒ thay đổi type của input thành type khác (VD: xem pass =D )
    • Nếu sửa thành type ko đúng ⇒ fallback về default
  • Note: khi code HTML hạn chế dùng những thẻ về styling <strong> , <i>, <ul> , <ol> , <li> vì khó quản lý ⇒ dùng CSS
  • Note: dùng CSS quyết định rule CSS nào đc apply khi load web (mobile/ PC)
