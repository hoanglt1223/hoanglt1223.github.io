Các thuộc tính của đối tượng XMLHttpRequest

Thuộc tínhMô tả
onReadyStateChangeNó được gọi bất cứ khi nào thuộc tính readystate bị thay đổi. Nó không được sử dụng với các yêu cầu đồng bộ.
readyStateĐại diện cho trạng thái của yêu cầu. Nó nằm trong khoảng từ 0 đến 4. 0 UNOPENED phương thức open() không được gọi. 1 OPENED open() được gọi nhưng send() không được gọi. 2 HEADERS_RECEIVED send() được gọi, và các tiêu đề và trạng thái có sẵn. 3 LOADING Tải xuống dữ liệu; responseText giữ dữ liệu. 4 DONE Thao tác hoàn tất.
reponseTextTrả về phản hồi dưới dạng văn bản.
responseXMLTrả về phản hồi dưới dạng XML.
  • Ready State: 0 ⇒ 4
    • 0: tàu đc tạo nhưng chưa chất hàng
    • 1: chưa chất hàng
    • 2: tàu đã đc gửi đi
    • 3: đang đợi tàu về
    • 4: done (chuyến hàng kết thúc)

  • Nhận biết khi gửi request bị lỗi ⇒ dùng status code, VD: 4xx, 5xx, 2xx
    • 404: Not Found ⇒ đúng url method rồi nhưng ko có gì trả về hết
    • 401: Unauthorized ⇒ lỗi định danh, ko xác định đc request ai gửi, kẹt ở lớp định danh
    • 403: Forbidden ⇒ request có authenication hoặc ko có authentication nhưng bị deny
    • 400: Bad Request ⇒ ko định nghĩa http url hay method
    • 500: Internal Server Error ⇒ đang quá trình xử lí dữ liệu của server mà có lỗi thì là lỗi 500
    • 502: Bad Gateway ⇒ tạch ở cổng
    • 503: Service Unavailable ⇒ quá tải
    • 504: Gateway timeout
    • 200: OK =)) ⇒ mọi thứ ổn ko vấn đề
