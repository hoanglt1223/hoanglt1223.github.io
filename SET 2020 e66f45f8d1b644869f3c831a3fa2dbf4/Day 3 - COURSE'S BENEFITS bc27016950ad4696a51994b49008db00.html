Day 3 - COURSE'S BENEFITS

Benefit từ khoá này.

  1. MINDSET (Tư duy lập trình)
  • Tự giải quyết vấn đề.

2. Teamwork (Rất quan trọng)

  • Cái cần thiết nhất khi học, làm.

3. Environment (Môi Trường)

  • Cung cấp những thứ cần thiết để có thể áp dụng thực tế những gì đã học

4. Philosophy (Triết học)

  • VD: Bất kì thư viện nào ra đời cũng dùng cho 1 mục đích nhất định => phải hiểu mục đích để sử dụng đúng cách

5. Usual lib / framework

  • VD: JavaScript (Dễ hiểu, sử dụng)

6. Code Review

  • Giúp biết sai chỗ nào để cải thiện.

7. VD công việc

  • Dev: chỉ code ko cần mấy kia (80% ở VN)
  • Software Engineer (Khoá này dạy) (Lương cao hơn Dev) (20%) -> Ko cắm đầu vào code, phải biết design.
  • Thực tế: + Tech Lead thiết kế architecture. + Senior (SE) viết, vẽ Doc & FLow (ERD, Usecase, Class, Sequence, Activity, Test case). + Dev( nhìn vào Doc xong code)

8. Not Provide

  • Experience: có những cái k thể dạy phải tự khám phá
  • Lib/ framework: có những cái mới phải tự học

9. Study Experience

  • Tools
  • Design > Code: Vẽ flow, vẽ wireframe (nhanh hơn). Không nên code tiến lên (code tới đâu tính tới đó). + I. ERD, usecase, activity, sequence. + II. Mỗi ng nhìn vào diagram sẽ có nhu cầu khác nhau: Dev, PO, QC => ai cũng phải đọc để hiểu cơ bản về ứng dụng cần làm => ko bị sai lệch về hướng phát triển. + III. Quy trình làm việc: Todo > In Progress > Review > Testing > Fixing > Done. + IV. Estimate Est 5h 4h (mũi tên xuống) Real 3h 4h (nt) KPI giảm (nt)
