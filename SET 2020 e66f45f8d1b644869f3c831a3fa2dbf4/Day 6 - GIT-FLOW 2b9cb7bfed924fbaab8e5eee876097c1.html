Day 6 - GIT-FLOW

Buổi 6 - 03/07/2020

Stash (lưu code) (đem cất đi)

  • Khi cần checkout qua nhánh khác có thể xảy ra trường hợp ko checkout dc
  • Khi tạo 1 file mới sẽ có trạng thái untracked => theo dõi đã tồn tại trên cây git hay chưa => ko thể conflict
  • Stash có HASH, file changes , base (giống commit) nhưng chỉ trên local kbh có trên origin
  • Working directory: những file mình đã có tác động
  • Khi nhấn stash sẽ tạo 1 cục stash (giống commit)
  • Rebase vẫn còn base của stash (kể cả force-push). Commit là base

Merge request:

  • Chỉ làm trên origin, k làm trên local (Ở đây là trên gitlab or trên trang host repo của mình)

Gitflow:

  • Step 1: Tạo nhánh => convention (tạo luật, VD: push lúc nào, ở đâu)
  • Step 2: Code/ change
  • Step 3: Stage -> commit -> push
  • Step 4: Rebase

    + 4.1: Checkout master + 4.2: Fetch + 4.3: Pull + 4.4: Rebase + 4.5: Resolve conflict + 4.6: Force Push

  • Step 5: Tạo Merge Request
  • Step 6 (step phụ): Khi có bug quay lại step 2 để fix và báo Lead (ko cần tạo merge request nữa vì đã có)

VD: Convention - Tên: GroupName/ Mã task - Feature/VDES-100
